**INCOMING**: Epic requirements (Container Test Infrastructure)
**OUTGOING**: Working `arni dev` CLI command for managing dev containers

## Description
Implement `arni dev` subcommand in arni-cli that manages development database containers using the existing compose.yml and podman. This replaces the testcontainers-rs approach with a more flexible CLI-based solution.

## Technical Approach
- Use existing `compose.yml` with all 6 databases (PostgreSQL, MySQL, MongoDB, SQL Server, Oracle, DuckDB N/A)
- Integrate with podman/podman-compose for container management
- Provide commands: start, stop, status, logs, clean
- Check container health and readiness
- Environment-based configuration support

## Acceptance Criteria
- [ ] `arni dev start` starts all or specific database containers
- [ ] `arni dev stop` stops running containers
- [ ] `arni dev status` shows container health and connection info
- [ ] `arni dev logs [service]` shows container logs
- [ ] `arni dev clean` removes containers and volumes
- [ ] Podman/podman-compose integration working
- [ ] Health checks verify containers are ready
- [ ] Connection strings displayed for ready containers
- [ ] Documentation in README for dev container usage

## Commands Design
```bash
arni dev start [postgres|mysql|mongodb|mssql|oracle|all]
arni dev stop [service|all]
arni dev status
arni dev logs [service]
arni dev clean [--volumes]
```
